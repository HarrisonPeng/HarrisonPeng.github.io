3.input默认宽度

input元素是有默认占位字符数的，数量在每个浏览器具体表现不同，比如在chrome中，好像就是默认占20个字符的宽度，这个解决起来也特别简单，我们可以通过给input增加width来解决其默认宽度过长的问题。

4. Webpack output path

在将一个新的前端项目部署到服务器上的时候，项目对于静态资源的引用出现了一些问题。这个项目前端是基于Vue全家桶，使用Webpack进行打包的SPA，Node中间层基于Koa2搭建。前端的静态资源使用koa-static部署到服务器。在访问域名时，ngnix会将请求转发到服务器的8080端口，然后我们将node运行在这一端口上，再将koa-router的/匹配置空。这样在koa-router未匹配到对应的路由/时就会将请求交给koa-static中间件，从而实现对于网页的加载。但是在项目打包上线时遇到一个问题：网页对于静态资源（Css、Javascript）等的引用路径为相对路径，而我们想要绝对路径。这样在切换URL的时候才能够保持加载同样的引用。最后发现是由于Webpack配置中的output的publicPath写错导致的。

output中的path和publicPath非常容易搞混：

  • path：所有通过webpack打包文件的输出路径
  • publicPath：解析引用文件的路径，简单的来说就是引用文件的URL相对于HTML文件的路径

5.微信小程序安卓图片显示模糊

微信小程序安卓端，如果图片的尺寸过大、过长，微信就会自动压缩该图片。目前没有特别好的解决办法，现在的做法是将大图裁剪为小图，然后再显示的时候按照顺序无缝拼接。
