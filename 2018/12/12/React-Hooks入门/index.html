React Hooks入门

发表于 2018-12-12

React Hook的出现主要是为了解决wrapper hell和对Class Component进行解耦，他能够帮组我们写出更加清晰的代码。在hooks出现之前，我们在编写React Class Component时不可避免的要在constructor里面声明一大堆state、编写各种控制state内部变量的function和在各种life circle里对整体流程进行控制。然后，这样做的坏处是我们不可避免的要在constructor里面对class内部的function进行bind以改变他的作用域，当然也可以在定义function时使用箭头函数。我们还需要在componentWillUnmount时去对我们订阅的函数进行解绑。而在我们使用react之前倡导的HOC即高阶函数对另外的函数进行包装时，实际业务中最终的代码往往会变得非常难看，因为过多的嵌套，导致代码的可读性下降，例如redux的provider等等。而hooks（useContext）的出现使得我们可以减少大量的nest，并且使得我们最终的代码更加functional，更加容易阅读。

useEffect在每次渲染之后（包括第一次渲染）都会执行，我们必须在组件中调用，因为它总是依赖于State内部状态。
当然React也为useEffecthook提供了相应的清理函数，即在相应的useEffect函数中返回一个函数，React将在组件卸载时去执行它。
在之前的版本中我们为了节省内存一般是在componentDidMount中去进行订阅，最后再componentWillUnmount中去取消订阅，而在使用了useEffect后就不需要将这些代码分开了。
useEffect的出现应该是为了减少对于life circle的使用，因为它更见简便，而且易读性更好。
