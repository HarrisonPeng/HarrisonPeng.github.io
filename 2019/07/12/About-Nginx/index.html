About Nginx

发表于 2019-07-12

跨域

  • 拦截路径/api，也可以改为任意正则匹配
  • 添加跨域相关请求头
  • proxy_set_header 允许添加或更改传给代理服务器的请求头
  • $http_host、$remote_addr、$scheme为Nginx内置变量
  • rewrite可以将原本的路径重写，因为我们在Nginx中只会去拦截需要配置跨域的请求(一般用/api作为标识)，所以通过Nginx链接后端服务器时，实际上不需要这一段标识，所以通过rewrite将其删除
  • break 继续此次请求后续的处理
  • proxy_pass 代理服务器，例如我们可以将前端的请求代理到服务器localhost:9000，然后再通过服务器向

location 匹配

  1. 从location配置列表由上至下开始查找前缀匹配的location，每次查找时记录匹配到的最长字符串
  2. 如果精准匹配，停止查找，使用该条配置
  3. 如果没有精准匹配，则开始查找正则匹配的location，如果匹配，停止查找，使用该配置
  4. 如果都没有匹配到，则使用最长匹配的前缀匹配location

前缀匹配可以配置成乱序的，但是正则匹配的顺序很重要，因为是从上至下开始匹配的，一旦匹配到合适的正则就会停止查找

  • 请求/精准匹配 A，不再往下查找。
  • 请求/index.html匹配 B。首先查找匹配的前缀字符，找到最长匹配是配置 B，接着又按照顺序查找匹配的正则。结果没有找到，因此使用先前标记的最长匹配，即配置 B。
  • 请求/user/index.html匹配 C。首先找到最长匹配 C，由于后面没有匹配的正则，所以使用最长匹配 C。
  • 请求/user/1.jpg匹配 E。首先进行前缀字符的查找，找到最长匹配项 C，继续进行正则查找，找到匹配项 E。因此使用 E。
  • 请求/images/1.jpg匹配 D。首先进行前缀字符的查找，找到最长匹配 D。但是，特殊的是它使用了^~修饰符，不再进行接下来的正则的匹配查找，因此使用 D。
  • 请求/documents/about.html匹配 B。因为 B 表示任何以/开头的 URL 都匹配。在上面的配置中，只有 B 能满足，所以匹配 B。

均衡负载

Nginx的均衡负载目前一般使用upstream模块配合实现

负载均衡策略：

  • 轮询：每次讲当前请求转发到上一个请求所转发的下一个服务器，如果服务器crash，可以自动剔除
  • weight：可以为每台服务器配置weight，实际还是使用轮询策略，但是可以降低性能较低服务器的weight，以减少其被访问量
  • ip_hash：每个请求按照请求的ip地址来分发到不同的服务器，这样相同ip的请求就能固定访问同一台服务器
  • backup：备用服务器，当所有非backup机器crash或者繁忙的时候才会来访问这台机器
  • down：表示当前机器不会参与负载
  • fair：根据服务器的响应时间来自动确定wieght
